# HeiGe-PPT（演讲锻造系统）

PPT 不是写出来的，是讲出来的。

大多数 AI 做出来的 deck，是把要点堆成一面字墙：标题 + 三个 bullet，一页接一页，母版一个调子到底。投到屏幕上，观众一边听你念，一边走神。讲完了，一页都记不住。

HeiGe-PPT 换一种做法：**把每一页讲成一句话，把整套 deck 导成一场演出。** 每一页只有一个观点、一个主角、一个让人记住的瞬间。讲完之后，观众能复述出你的主线，记得住你那几页。

这套方法论的目标只有一个：**让你做出的 deck 有气场、有记忆点、站在台上敢讲、讲完有人记得。**

---

## 招牌内核：导一场演出，不是排一摞幻灯片

做 deck 之前，先想清楚一件事：**观众从第一页到最后一页，你想带他走一段什么样的路、最后让他做一个什么动作？**

排幻灯片的思维问的是「这次汇报要讲哪几块内容」。导演出的思维问的是「主线是什么、高潮在哪、观众走的时候记住了哪一句、要他干什么」。

五道工序，是这套思维的具体落地。每套 deck 都要过一遍。

### 1. 一页一句话（One Sentence per Slide）

每一页只讲**一个**观点，这个观点能用一句话说清楚。

- 一页只有一个主角观点。讲不完的，拆成两页，别挤在一页里。
- 反对「标题 + 三个 bullet + 每个 bullet 两行字」这种字墙。那是发给人看的文档，不是站在台上讲的 deck。
- 一个判断：如果这一页的内容你没法用一句话概括，说明这页没想清楚，回去拆。
- 正文是给你讲的，不是给观众读的。观众读字的时候就不听你讲了。字越少，注意力越在你身上。

> 自检：把每一页的核心观点抄成一句话，连起来读一遍。如果这串句子本身就是一条清楚的主线，deck 的骨架就立住了。

### 2. 黄金三秒（The 3-Second Rule）

每一页投出来，观众扫一眼，**3 秒内**要 get 到这页要说什么。

- 每页有且只有一个视觉主角：一句大字、一个大数字、一张大图、一个对比。其余全部退后。
- 主角要够大、够狠。台下最后一排也能看清，才算合格。投影仪不是显示器，字号要敢往大了上。
- 信息层级靠尺寸和位置拉开，不靠颜色堆。一页里别有三个东西在抢「我最重要」。

### 3. 节奏编排（Deck Pacing）

整套 deck 像一场演出，有起承转合：**开场抓人 → 摆出痛点 → 给出转折 → 论证铺陈 → 推向高潮 → 收束行动**。

- 不要每一页一个调子、一个版式。那是模板套壳，不是演出。
- 不同功能的页面要有不同的视觉重量：封面页立气场，金句页留白给冲击，数据页给密度，过渡页给喘息，收尾页给行动。
- 强页之后给一页轻的，让观众喘口气、消化。连续五页全是密集数据，观众就睡了。
- 收尾要干脆：一个明确的行动号召，或一句记得住的话。别用「谢谢聆听」四个字草草收场。

各类 deck 的节奏模板见 `references/deck-pacing-templates.md`。

### 4. 视觉锤（Visual Hammer）

最关键的几个观点，每个配一个**让人记住的视觉**，把观点像钉子一样砸进观众脑子。

- 可以是：一个超大数字、一张强对比图、一个比喻图形、一条让人倒吸气的趋势线、一句占满整屏的宣言。
- 视觉锤不需要多，整套 deck 有三五个就够。但你最想让人记住的观点，必须有一个。
- 纯文字的页面记不住。能用一个数字、一张图、一个图形说清楚的，就别用一段话。

> 自检：讲完合上电脑，问自己「观众最可能记住哪三页」。答得上来，且那三页正好是你最想传达的，视觉锤就到位了。

### 5. 反 AI 体检（Anti-Slop Check）

出货前，强制过一遍体检，把「AI 味」和「字墙味」扼杀在上台前。

最常见的烂 deck 特征（命中任何一条都要回去改）：
- 标题 + 三个等长 bullet，整套 deck 每页都这个结构
- 母版套壳：每页一样的版式、一样的配色、一样的节奏，从头平铺到尾
- 一页塞两三个观点，没有单页焦点
- 通篇纯文字，没有一个视觉锤，没有一页能被记住
- 字小、行密，台下最后一排看不清
- 用 emoji 当图标撑场面、用「谢谢聆听」草草收尾

完整清单见 `references/anti-slop-checklist.md`。**这一步不是可选的，是交付门槛。**

## 使用流程

### 第一步：定调（动手前必做）

写任何代码之前，先和用户对齐三件事。根据平台能力选合适的交互方式：

**问题 1：这套 deck 是什么场景、讲给谁、要他们干什么？**
- 场景：创业路演 / 产品发布 / 年度复盘 / 方案提案 / 技术分享 / 培训课件 / 述职汇报……
- 观众：投资人 / 客户 / 老板 / 同行 / 学员？他们在意什么、最烦什么？
- 行动：讲完你要他们做什么？投钱、签单、点头、记住一个概念？这决定收尾页。

**问题 2：气质方向？**（从 `references/aesthetic-directions.md` 选一个**极端**方向，别选「商务简约」这种正确的废话）
- 凶悍/工业、优雅/高定、克制/极简、科技/未来、张扬/极繁、温暖/人文……

**问题 3：有没有硬约束？**
- 必须包含的内容、品牌色、字体、页数范围、是否要导出 PDF、是否要演讲备注？

如果用户说「你定」，就**自己拍一个大胆的方向**，动手前用一句话告诉他你定了什么，给他喊停的机会。定调一次到位，别反复问。

### 第二步：搭主线（写每页一句话）

定好调，先把内容拆成「一页一句话」的主线：
- 列出每一页的核心观点，每个观点一句话。
- 把这串句子连起来读，确认它本身就是一条清楚、有起伏的主线。
- 这一步是骨架，骨架不对，做得再好看也是散的。

### 第三步：编排节奏，定视觉锤

- 按节奏曲线给每页定功能：哪页封面、哪页痛点、哪页高潮、哪页收尾。
- 标出最关键的三五个观点，给每个想一个视觉锤（大数字 / 对比 / 图形 / 宣言）。

### 第四步：写生产级代码

按 `references/deck-production-spec.md` 的硬规格做，要点：
- **单文件 HTML**，16:9 幻灯片，一次一页，零依赖（除 Google Fonts）。
- **键盘翻页**：← → 和空格翻页，可加页码和进度指示。
- **中文字体必须有系统兜底**（PingFang SC / Microsoft YaHei），Google Fonts 挂了也不崩。
- **可导出 PDF**：每页对应一页，打印友好。
- 字号往大了上，台下最后一排能看清；字体做性格，别用系统默认裸奔。
- **可编辑**：成品默认带上可编辑层（把 `references/editable-layer.md` 里的 script 块贴在 `</body>` 前），用户打开点「编辑」就能在浏览器里直接改幻灯片文字，改完导出 PDF 或下载独立 HTML；放映模式下方向键翻页和打字不打架，工具栏不进 PDF。

### 第五步：反 AI 体检 + 试讲复检

出货前强制自检：
1. 过一遍 `references/anti-slop-checklist.md`，命中的全部干掉。
2. 试讲复检：每页能一句话讲清吗？3 秒能看懂吗？主线连贯吗？有几个视觉锤？最后一排看得清吗？
3. 任何一项不过关，回去改，别交。

## 设计资产（references/）

- `aesthetic-directions.md`：气质方向库。每种气质给出色彩 / 字体 / 排版 / 封面 / 视觉锤的具体打法。
- `deck-pacing-templates.md`：节奏曲线模板。路演 / 产品发布 / 复盘 / 提案 / 分享各类 deck 的页面节奏排布。
- `deck-production-spec.md`：单文件 HTML 幻灯片的硬规格（尺寸 / 翻页 / 字体兜底 / 导出 / 性能），照着做不踩坑。
- `anti-slop-checklist.md`：反 AI 体检清单。出货前逐条过。
- `editable-layer.md`：可编辑层。一段拖进 `</body>` 前就生效的 script，让成品在浏览器里直接改幻灯片文字、导出 PDF/HTML，放映翻页和打字不打架、工具栏不进打印。

## 成品参考（examples/）

同一套方法论，跨场景、跨气质能产出完全不同的 deck。以下样例均用本方法论现场锻造，可直接打开、键盘翻页：

- `heige-pitch.html`：HeiGe-PPT 自己的产品路演 deck。凶悍/工业气质。
- `product-keynote.html`：AI 产品发布会 keynote。科技/未来气质。
- `annual-review.html`：团队年度复盘。克制/数据气质。
- `consulting-proposal.html`：给客户的方案提案。优雅/高定气质。
- `campus-talk.html`：一场技术分享演讲。张扬/人文气质。

## 一句话总结

**别人把要点堆成一摞幻灯片，你把观点导成一场演出。一页一句话，讲完有人记得。**
